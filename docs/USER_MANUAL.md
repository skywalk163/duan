## 快速开始

### 安装

```bash
# 克隆仓库
git clone https://github.com/your-repo/duan.git
cd duan

# 运行全部测试（推荐）
python tests/run_tests.py

# 运行快速验证
python tests/run_tests.py --quick

# 仅运行边界测试
python tests/run_tests.py --edge

# 列出所有测试文件
python tests/run_tests.py --list
```

### 第一个程序

创建 `hello.duan` 文件：

```段言
打印"你好，世界！"。
```

编译并运行：

```bash
python cli/duanc.py hello.duan --run
```

输出：
```
你好，世界！
```

## 基础语法

### 注释

段言支持以下注释语法：

```段言
# 这是单行注释（推荐使用）
// 这也是单行注释
注：这也是注释（以"注"开头）
```

段言支持三种注释风格，可以灵活使用。

### 语句结束

每个语句以句号 `。` 结束。

```段言
定义甲等于123。
打印甲。
```

### 标点符号

| 中文符号 | 英文符号 | 用途 |
|---------|---------|-----|
| `。` | `.` | 语句结束 |
| `，` | `,` | 参数分隔、管道操作 |
| `：` | `:` | 块开始、类型注解 |
| `（` `）` | `(` `)` | 括号 |
| `【` `】` | `[` `]` | 列表 |
| `「` `」` | `"` `"` | 字符串 |

## 变量与数据类型

### 变量声明

**语法：** `定义 变量名 等于 值。`

```段言
定义年龄等于25。
定义姓名等于"张三"。
定义成绩等于95.5。
```

### 数据类型

**数字类型：**
```段言
定义整数等于123。
定义浮点等于3.14。
定义中文数字等于三加五。  # 8
```

**字符串：**
```段言
定义消息等于"你好世界"。
```

**布尔值：**
```段言
定义正确等于真。
定义错误等于假。
```

**列表：**
```段言
定义列表等于1，2，3，4，5。
```

### 算术运算

```段言
定义甲等于10。
定义乙等于3。

定义和等于甲加乙。      # 13
定义差等于甲减乙。      # 7
定义积等于甲乘乙。      # 30
定义商等于甲除乙。      # 3.333...
定义余等于甲取余乙。    # 1
```

### 比较运算

```段言
如果甲大于乙那么打印"甲大"。
如果甲小于乙那么打印"甲小"。
如果甲等于乙那么打印"相等"。
```

## 控制流程

### 条件语句

**基础语法：**
```段言
如果条件那么执行语句。
```

**示例：**
```段言
定义分数等于85。

如果分数大于等于90那么打印"优秀"。
否则如果分数大于等于80那么打印"良好"。
否则如果分数大于等于70那么打印"中等"。
否则打印"及格"。
```

### 循环语句

**遍历循环：**
```段言
定义列表等于1，2，3，4，5。

遍历列表中的元素：
  打印元素。
```

**当循环（while）：**
```段言
定义计数等于1。

当计数小于等于5：
  打印计数。
  定义计数等于计数加1。
```

**循环控制：**
```段言
遍历列表中的元素：
  如果元素等于3那么跳出。
  打印元素。
```

## 函数定义

### 基础函数

**语法：** `《函数名》段(参数)：函数体。`

```段言
《加法》段(甲, 乙)：
  返回甲加乙。

定义结果等于《加法》参数三和五。
打印结果。  # 8
```

### 递归函数

**阶乘：**
```段言
《阶乘》段(数)：
  如果数小于等于1那么返回1。
  返回数乘《阶乘》参数数减1。

定义结果等于《阶乘》参数5。
打印结果。  # 120
```

**斐波那契：**
```段言
《斐波那契》段(数)：
  如果数小于等于2那么返回1。
  返回《斐波那契》参数数减1加《斐波那契》参数数减2。

定义结果等于《斐波那契》参数10。
打印结果。  # 55
```

## 类与对象

### 类定义

**语法：**
```
类 类名：
  属性列表
  方法定义
类结束
```

**示例：**
```段言
类 学生：
  属性 姓名
  属性 年龄
  
  构造(姓名, 年龄)：
    定义己姓名等于姓名。
    定义己年龄等于年龄。
  
  介绍：
    打印"我叫"加己姓名加"，今年"加己年龄加"岁。"。
类结束

定义学生等于创建学生参数"张三"，20。
学生.介绍。
```

### 继承

```段言
类 大学生 继承 学生：
  属性 专业
  
  构造(姓名, 年龄, 专业)：
    父类构造(姓名, 年龄)。
    定义己专业等于专业。
  
  介绍：
    打印"我叫"加己姓名加"，专业是"加己专业加"。"。
类结束
```

### 属性访问

- `己属性` - self.属性
- `对象.属性` - 对象.属性

## 异常处理

### 基本异常处理

```段言
尝试：
  定义文件等于打开参数"data.txt"。
  定义内容等于文件.读取。
  打印内容。
捕获 文件错误 为 错误：
  打印"无法读取文件："加错误.信息。
```

### 多异常捕获

```段言
尝试：
  定义数值等于转换整数参数"abc"。
捕获 数值错误 为 错误：
  打印"不是有效的数字"。
捕获 其他错误 为 错误：
  打印"发生未知错误"。
最终：
  打印"处理完成"。
```

### 抛出异常

```段言
《检查年龄》段(年龄)：
  如果年龄小于零：
    抛出 数值错误 参数"年龄不能为负数"。
  返回年龄。
```

## 编译器使用

### CLI 工具

**基本用法：**
```bash
# 编译文件
python cli/duanc.py file.duan

# 编译并运行
python cli/duanc.py file.duan --run

# 指定输出文件
python cli/duanc.py file.duan -o output.py

# 显示 Token 流
python cli/duanc.py file.duan --tokens

# 显示 AST
python cli/duanc.py file.duan --ast

# 详细输出
python cli/duanc.py file.duan -v

# 创建示例项目
python cli/duanc.py --init
```

### Python API

```python
from lexer import Lexer
from duan_parser_v3 import DuanParser
from semantic_analyzer import SemanticAnalyzer
from code_generator import PythonCodeGenerator

# 编译流程
lexer = Lexer()
parser = DuanParser()
analyzer = SemanticAnalyzer()
generator = PythonCodeGenerator()

# 编译代码
source = '定义甲等于123。'
tokens = lexer.tokenize(source)
module = parser.parse(source)
analyzer.analyze(module)
python_code = generator.generate(module)

print(python_code)
```

```段言
《计算器》段(操作, 甲, 乙)：
```

### 3. 银行账户

```段言
类 银行账户：
  属性 账户号
  属性 余额
  
  构造(账户号, 初始余额)：
    定义己账户号等于账户号。

    返回己余额。

账户.取款参数200。
```

## 常见问题

### 1. 为什么我的代码编译失败？

**常见原因：**
- 缺少句号 `。`
- 关键字拼写错误
- 括号不匹配
- 字符串引号不匹配

**解决方法：**
使用 `--tokens` 查看词法分析结果：
```bash
python cli/duanc.py file.duan --tokens
```

### 2. 如何调试代码？

**方法1：** 使用 `--ast` 查看语法树
```bash
python cli/duanc.py file.duan --ast
```

**方法2：** 使用 `-v` 详细输出
```bash
python cli/duanc.py file.duan -v
```

### 3. 段言支持哪些关键字？

**核心关键字：**
- 定义：`定义`、`常量`、`类型`
- 条件：`如果`、`那么`、`否则`
- 循环：`遍历`、`当`、`跳出`、`跳过`
- 函数：`段`、`返回`
- 类：`类`、`继承`、`属性`、`构造`
- 模块：`模块`、`导入`、`导出`
- 异常：`尝试`、`捕获`、`抛出`、`最终`

### 4. 如何参与开发？

**步骤：**
1. Fork 项目仓库
2. 创建功能分支
3. 编写代码和测试
4. 提交 Pull Request

**开发指南：**
- 代码风格：遵循 PEP 8
- 测试要求：新增功能必须有测试
- 文档要求：新增功能必须更新文档
