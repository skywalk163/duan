# Duan Level 6 设计规格：无空格分词 + 纯缩进 + 类型注解

**版本**: v6.0
**日期**: 2026-07-01
**状态**: 待审查
**基于**: Duan Level 5（异常处理 + 模块系统）

---

## 一、设计目标

Level 6 是段言语法的重大升级，核心目标是让段言真正成为"**语句内无空格、语句间用缩进**"的自然中文编程语言。

### 三大核心特性

1. **无空格分词**：语句内关键字与标识符紧密相连，词法分析通过最长前缀匹配自动拆分
2. **纯缩进语法**：彻底移除 `结束` 关键字，用缩进层级表示块结构
3. **类型注解**：可选类型系统，服务于 LLVM 后端类型推断和可选运行时检查

### 设计原则

- **向后兼容**：所有 Level 5 语法（有空格、有结束）在 Level 6 中仍然有效
- **渐进采用**：新语法是可选的，用户可以逐步迁移
- **零开销默认**：类型注解默认不产生运行时代码，仅用于编译时分析
- **中文优先**：所有语法元素都有中文形式，符号/英文是别名

## 二、无空格分词

### 2.1 原理

词法分析采用**最长前缀匹配**策略：

1. 读取连续的中文字符串（含字母、数字、下划线）
2. 从最长关键字开始，尝试匹配字符串前缀
3. 匹配成功则拆出关键字 token，剩余部分继续匹配
4. 全部匹配失败则整个字符串作为标识符返回

### 2.2 支持无空格的关键字

Level 6 初始支持以下关键字的无空格分词（约 45 个）：

| 类别 | 关键字 |
|------|-------|
| 声明 | `设`、`为`、`设为`、`类型`、`常量` |
| 函数 | `段`、`段落`、`接收`、`返回`、`结束`（兼容用） |
| 类 | `类`、`属性`、`己`、`继承`、`父` |
| 控制流 | `如果`、`则`、`否则`、`否则如果`、`当`、`遍历`、`在`、`跳出`、`继续` |
| 异常 | `尝试`、`捕获`、`最终`、`抛出` |
| 模块 | `导入`、`导出` |
| 字面量 | `真`、`假`、`空` |
| 比较运算 | `等于`、`不等于`、`小于`、`大于`、`小于等于`、`大于等于` |
| 逻辑运算 | `且`、`或`、`非` |
| 算术运算 | `加`、`减`、`乘`、`除`、`取模` |

### 2.3 转义机制

反引号 `` ` `` 包裹的内容强制作为标识符，用于极少数需要用关键字做变量名的场景。

```
`真` = 10          // 反引号转义，'真' 作为变量名
设 `类` 为 "test"  // '类' 作为变量名
```

### 2.4 分词示例

| 输入 | 分词结果 |
|------|---------|
| `设x为10` | `设`(KW) + `x`(ID) + `为`(KW) + `10`(NUM) |
| `如果x大于0则返回x` | `如果`(KW) + `x`(ID) + `大于`(KW) + `0`(NUM) + `则`(KW) + `返回`(KW) + `x`(ID) |
| `段加法接收a,b返回整数：` | `段`(KW) + `加法`(ID) + `接收`(KW) + `a`(ID) + `,`(COMMA) + `b`(ID) + `返回`(KW) + `整数`(ID) + `：`(COLON) |
| `设整数列表为[]` | `设`(KW) + `整数列表`(ID) + `为`(KW) + `[`(LBRACKET) + `]`(RBRACKET) |

> **说明**：`整数列表` 会被识别为一个标识符（因为 `整数` 不是关键字）。类型识别在语法分析阶段进行（见第三章）。

## 三、类型注解

### 3.1 变量声明语法

| 语法形式 | 说明 | 示例 |
|---------|------|------|
| `设 变量 为 类型` | 仅类型标注，无初始值 | `设 x 为 整数` |
| `设 变量 为 类型 = 值` | 类型 + 初始值 | `设 x 为 整数 = 10` |
| `设 变量 = 值` | 类型推断 | `设 x = 10` |
| `设 变量 为 值` | 旧语法（类型推断，兼容） | `设 x 为 10` |
| `变量 设为 值` | 重新赋值 | `x 设为 20` |

### 3.2 函数定义语法

```
段 函数名 接收 参数1 类型1, 参数2 类型2 返回 返回类型：
    函数体
```

示例：
```
段加法接收a整数,b整数返回整数：
    返回a加b
```

### 3.3 类型体系

#### 3.3.1 基础类型

| 类型名 | 对应 Python 类型 | 说明 |
|-------|----------------|------|
| `整数` | int | 整数 |
| `小数` | float | 浮点数 |
| `文本` | str | 字符串 |
| `布尔` | bool | 布尔值 |
| `空` | NoneType | 空值 |

#### 3.3.2 复合类型

**标准写法（方括号泛型）**：
```
列表[整数]           // 整数列表
字典[文本, 整数]      // 文本到整数的字典
列表[列表[整数]]      // 嵌套列表
```

**中文语法糖**：
```
整数列表             // 等价于 列表[整数]
文本到整数字典        // 等价于 字典[文本, 整数]
整数列表列表          // 等价于 列表[列表[整数]]
```

#### 3.3.3 类类型

用户定义的类名直接用作类型：
```
设用户为用户信息      // 用户信息 是类名
```

#### 3.3.4 函数类型

**中文风格**：
```
接收整数返回文本       // 接收一个整数，返回文本的函数
接收整数,小数返回布尔   // 接收整数和小数，返回布尔的函数
```

**箭头风格**：
```
整数 -> 文本          // 等价于 接收整数返回文本
(整数, 小数) -> 布尔   // 等价于 接收整数,小数返回布尔
```

### 3.4 类型识别机制

类型名**不作为关键字**（避免命名歧义），在语法分析阶段识别：

1. **变量声明**：在 `设` 和 `为` 之间，尝试最长类型前缀匹配
2. **函数参数**：参数名后紧跟的标识符，检查是否为类型名
3. **函数返回**：`返回` 后面、`：` 前面的标识符

### 3.5 运行时类型检查

- **默认关闭**：零性能开销
- **编译选项开启**：`duan build --runtime-checks file.duan`
- **语句级控制**：
  ```
  开启类型检查
  // 以下代码会做类型检查
  关闭类型检查
  // 以下代码不做类型检查
  ```
- **检查失败**：抛出 `类型错误`（TypeError）

## 四、纯缩进语法

### 4.1 核心规则

- 块结构通过 **缩进层级** 隐式标记开始和结束
- 块开始用 `：` 标记
- 缩进增加 → 进入新块（INDENT）
- 缩进减少 → 退出当前块（DEDENT）
- `结束` 关键字保留为兼容语法，不再必需

### 4.2 缩进规范

- **缩进字符**：仅空格
- **缩进宽度**：不固定，只要同一文件内层级一致即可
- **混合缩进**：不允许，报错

### 4.3 单行块

`：` 后可以直接跟语句（单行块）：
```
如果x大于0：返回x
当n>0：n设为n减1
```

`则` 关键字是可选语法糖，与 `：` 等价（仅用于 `如果` 语句中，提升可读性）：
```
如果x大于0则返回x          // 等价于 如果x大于0：返回x
如果x大于0则：返回x         // 则 + 冒号也可以（则是冗余的）
```

限制：
- `否则`/`否则如果`/`捕获`/`最终` 必须另起一行，与前一块同级缩进
- 单行块内只能有一条语句
- `则` 仅用于 `如果` 语句，其他块结构不支持

### 4.4 示例

#### 条件语句
```
如果x大于0：
    输出("正")
否则如果x小于0：
    输出("负")
否则：
    输出("零")
```

#### 循环
```
当i小于10：
    输出(i)
    i设为i加1
```

#### 函数
```
段斐波那契接收n整数返回整数：
    如果n小于2：返回n
    设a为整数=0
    设b为整数=1
    当n大于1：
        设c为整数=a加b
        a设为b
        b设为c
        n设为n减1
    返回b
```

#### 异常处理
```
尝试：
    x除0
捕获值错误as e：
    输出("值错误")
捕获：
    输出("其他错误")
最终：
    输出("清理")
```

#### 类
```
类动物：
    属性名字
    属性年龄
    段初始化接收名字文本,年龄整数：
        己.名字设为名字
        己.年龄设为年龄
    段说话：
        输出(己.名字 + "叫了一声")
```

## 五、运算符别名

### 5.1 别名表

词法分析阶段将所有别名统一转换为内部标记，后续编译阶段无差异。

| 内部标记 | 中文 | C符号 | 英文 | 优先级 |
|---------|------|-------|------|--------|
| `OR` | 或 | `\|\|` | `or` | 最低 |
| `AND` | 且 | `&&` | `and` | ↓ |
| `NOT` | 非 | `!` | `not` | ↓ |
| `EQ` | 等于 | `==` | | ↓ |
| `NEQ` | 不等于 | `!=` | | ↓ |
| `LT` | 小于 | `<` | | ↓ |
| `GT` | 大于 | `>` | | ↓ |
| `LE` | 小于等于 | `<=` | | ↓ |
| `GE` | 大于等于 | `>=` | | ↓ |
| `ADD` | 加 | `+` | | ↓ |
| `SUB` | 减 | `-` | | ↓ |
| `MUL` | 乘 | `*` | | ↓ |
| `DIV` | 除 | `/` | | ↓ |
| `MOD` | 取模 | `%` | | 最高 |

### 5.2 赋值运算符

| 内部标记 | 符号 | 说明 |
|---------|------|------|
| `ASSIGN` | `=` | 赋值 |
| `SET_AS` | 设为 | 中文赋值（变量在前） |

## 七、向后兼容策略

### 7.1 语法兼容

- 所有 Level 5 语法在 Level 6 中仍然有效
- 有空格的写法（`设 x 为 10`）和无空格的写法（`设x为10`）都支持
- 有 `结束` 的写法和纯缩进的写法都支持

### 7.2 迁移路径

用户可以逐步采用新语法，无需一次性全部修改。编译器自动识别两种风格。

## 八、实现路线

### 8.1 模块划分

1. **词法分析器改造**：最长前缀匹配、缩进 token 生成、运算符别名统一
2. **语法分析器改造**：纯缩进解析、类型注解解析、单行块支持
3. **类型检查器**（可选运行时）：运行时类型断言生成
4. **代码生成器改造**：适配新 AST 结构
5. **自举验证**：用 Level 6 语法重写编译器并自举

### 8.2 验证标准

- 所有 Level 5 测试用例继续通过（向后兼容）
- 新增无空格语法测试通过
- 新增纯缩进语法测试通过
- 新增类型注解测试通过
- 编译器可以用 Level 6 语法重写并成功自举
