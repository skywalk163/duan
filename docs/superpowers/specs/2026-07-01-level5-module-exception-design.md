# Duan Level 5 设计规格：模块系统与异常处理

**版本**: v5.0
**日期**: 2026-07-01
**状态**: 设计完成，待实现
**基于**: Duan Level 4（完整面向对象支持）

---

## 一、设计目标

Level 5 在 Level 4 的基础上增加两大核心特性：

1. **模块系统**：支持代码组织与复用，用户代码可拆分为多个 `.duan` 文件
2. **异常处理**：支持结构化的错误处理，提升复杂程序的可靠性

**核心约束**：
- 自举编译器保持单文件（避免"鸡生蛋"问题）
- 纯缩进语法，无 `结束` 关键字
- 与 Python 后端和 LLVM 后端兼容

## 二、模块系统设计

### 2.1 语法规范

#### 导入语句

```
导入 模块名
导入 子目录/模块名
```

**语法规则**：
- `导入` 后接标识符或路径（无引号）
- 路径使用 `/` 分隔（类似 Python import）
- 不支持 `from ... import ...` 语法（保持简洁）

**示例**：
```
导入 数学工具
导入 模块/字符串处理
导入 文件系统
```

#### 导出语句

```
# 方式一：独立导出语句（文件末尾）
导出 函数1
导出 函数2
导出 变量1

# 方式二：内联导出（函数定义时）
段 函数1 导出 接收 参数：
    ...

段 函数2 导出 接收 参数：
    ...
```

**语义**：
- `导出` 标记符号为公开，可被其他模块导入使用
- 未标记 `导出` 的符号为模块私有
- 内联 `导出` 和独立 `导出` 语句效果相同

### 2.2 模块搜索路径

当遇到 `导入 模块名` 时，编译器按以下顺序搜索：

1. **当前目录**：`./模块名.duan`
2. **当前目录子文件夹**：`./模块/模块名.duan`
3. **标准库目录**：`stdlib/模块名.duan`
4. **报错**：模块未找到

**路径解析规则**：
- `导入 数学工具` → 搜索 `数学工具.duan`
- `导入 模块/字符串处理` → 搜索 `模块/字符串处理.duan`
- 相对路径基于当前编译文件所在目录

### 2.3 编译机制

**内联编译**：
- `导入` 语句在编译时将目标模块代码内联到当前文件
- 生成代码为单一 Python 文件，无外部依赖
- 内联顺序按依赖关系拓扑排序（先编译被依赖模块）

**符号管理**：
- 导入模块的 `导出` 符号可在当前模块使用
- 私有符号（未 `导出`）不可访问
- 符号命名冲突：后导入模块覆盖同名符号（警告）

**循环导入检测**：
- 编译时检测循环依赖，报错并终止
- 错误信息：`检测到循环依赖: 模块A → 模块B → 模块A`

### 2.4 新增关键字

| 关键字 | 用途 |
|--------|------|
| `导入` | 导入模块 |
| `导出` | 导出符号 |

**模块系统新增关键字**：2 个（Level 4 的 34 个 + 2 = 36 个，异常处理再新增 4 个后总计 40 个）

## 三、异常处理设计

### 3.1 语法规范

#### 尝试-捕获-最终语句

```
尝试：
    可能出错的代码()

捕获 值错误 as e：
    输出("值错误：" + 字符串(e))

捕获 类型错误：
    输出("类型错误")

捕获：
    输出("其他错误")

最终：
    清理资源()
```

**语法规则**：
- `尝试：` 后接缩进块（受保护代码）
- `捕获 类型 as 变量：` 匹配指定类型异常
  - `as 变量` 可选，用于获取异常对象
  - 类型可省略，匹配所有异常
- 支持多重 `捕获` 块，按顺序匹配
- `最终：` 可选，块内代码始终执行

**生成 Python 代码**：
```python
try:
    可能出错的代码()
except ValueError as e:
    print("值错误：" + str(e))
except TypeError:
    print("类型错误")
except:
    print("其他错误")
finally:
    清理资源()
```

#### 抛出语句

```
抛出 "错误消息"
抛出 错误对象
抛出 变量名
```

**语义**：
- 抛出字符串 → 自动包装为 `Exception(字符串)`
- 抛出对象 → 直接抛出（需为 `BaseException` 子类）
- 抛出变量 → 直接抛出变量值

**生成 Python 代码**：
```python
# 抛出 "错误消息"
raise Exception("错误消息")

# 抛出 错误对象
raise 错误对象
```

### 3.2 异常类型映射

**内置中文别名**：

| 中文别名 | Python 类型 | 说明 |
|---------|-------------|------|
| 异常 | Exception | 通用异常基类 |
| 值错误 | ValueError | 值不正确 |
| 类型错误 | TypeError | 类型不匹配 |
| 键错误 | KeyError | 字典键不存在 |
| 索引错误 | IndexError | 列表索引越界 |
| 除零错误 | ZeroDivisionError | 除数为零 |
| 属性错误 | AttributeError | 属性不存在 |
| 名称错误 | NameError | 变量未定义 |
| 文件错误 | FileNotFoundError | 文件不存在 |
| 运行错误 | RuntimeError | 运行时错误 |
| 停止迭代 | StopIteration | 迭代结束 |

**使用规则**：
- 中文别名和英文原名都支持
- `捕获 ValueError` 和 `捕获 值错误` 效果相同
- 编译器内置映射表，无需用户定义

### 3.3 异常处理流程

```
源码：尝试 ... 捕获 值错误 as e：...
         ↓
    前端解析（统一）
         ↓
    生成异常处理 AST/IR
         ↓
    ┌────┴────┐
    ↓              ↓
Python 后端        LLVM 后端
try/except        setjmp/longjmp
```

**LLVM 后端兼容性**：
- 异常类型映射在代码生成阶段处理
- LLVM 后端使用 C 的 `setjmp/longjmp` 实现异常机制
- 中文别名映射为 C 结构体类型标识符

### 3.4 新增关键字

| 关键字 | 用途 |
|--------|------|
| `尝试` | 异常保护块开始 |
| `捕获` | 异常捕获块 |
| `最终` | 最终执行块 |
| `抛出` | 抛出异常 |

**Level 5 关键字总数**：40 个（Level 4 的 34 个 + 模块系统 2 个 + 异常处理 4 个 = 40 个，`as` 沿用英文不新增）

> 注：`as` 为英文关键字，非中文新增。

## 四、实现路线图

### 4.1 分阶段实施

| 阶段 | 内容 | 优先级 | 依赖 |
|------|------|--------|------|
| Phase 5.1 | 异常处理 | P0 | 无 |
| Phase 5.2 | 模块系统 | P1 | 无 |

| Phase 5.4 | 编译器模块化（可选） | P3 | Phase 5.2 |

### 4.2 Phase 5.1：异常处理

**任务清单**：
1. 词法分析：新增 `尝试`、`捕获`、`最终`、`抛出` 关键字
2. 语法解析：
   - `compile_try` 函数：解析 `尝试` 块
   - `compile_catch` 函数：解析 `捕获` 块（支持多重）
   - `compile_finally` 函数：解析 `最终` 块
   - `compile_throw` 函数：解析 `抛出` 语句
3. 异常类型映射表：内置中文别名 → Python 类型
4. 代码生成：生成 Python `try/except/finally` 代码
5. 测试：
   - 基本异常捕获
   - 多重捕获
   - 最终块执行
   - 抛出字符串和对象
   - 异常类型匹配

### 4.3 Phase 5.2：模块系统

**任务清单**：
1. 词法分析：新增 `导入`、`导出` 关键字
2. 语法解析：
   - `compile_import` 函数：解析 `导入` 语句
   - `compile_export` 函数：解析 `导出` 语句（独立 + 内联）
3. 模块解析器：
   - 搜索路径实现
   - 模块文件读取
   - 循环依赖检测
   - 拓扑排序
4. 代码生成：
   - 内联导入模块代码
   - 符号作用域管理
5. 测试：
   - 单模块导入
   - 多模块导入
   - 嵌套路径导入
   - 导出符号访问
   - 循环依赖检测

### 4.4 Phase 5.3：stdlib 迁移

**任务清单**：
1. 将 stdlib 从 Python 实现迁移到 Duan 实现
2. stdlib 文件添加 `导出` 标记
3. 用户代码可 `导入 stdlib/数学工具` 等

### 4.5 自举验证

**验证流程**：
```
Level 4 编译器
      ↓ 编译
bootstrap_level5.duan（单文件，支持异常处理）
      ↓ 编译
```

## 六、设计决策记录

### 6.1 模块系统

| 决策 | 选择 | 理由 |
|------|------|------|
| 导入语法 | `导入 模块名`（无引号） | 用户体验更自然 |
| 导出语法 | 内联 + 独立语句 | 灵活性最高 |
| 编译机制 | 内联编译 | 保持单文件、自举友好 |
| 搜索路径 | 当前目录 → 子目录 → stdlib | 符合直觉 |
| 自举编译器 | 保持单文件 | 避免"鸡生蛋"问题 |

### 6.2 异常处理

| 决策 | 选择 | 理由 |
|------|------|------|
| 语法风格 | `尝试`/`捕获`/`最终` | 与 Python 对应，语义清晰 |
| 异常类型 | 中文别名 + 英文原名 | 兼顾中文友好和 Python 兼容 |
| 捕获不带变量 | 允许 | 不关心具体错误时简化代码 |
| 抛出任意表达式 | 支持 | 字符串自动包装，对象直接抛出 |
| 多重捕获 | 支持 | 按类型分流处理 |
