# Duan Level 4 语法规范

**版本**: v4.0  
**日期**: 2026-06-30  
**状态**: ✅ 已实现并通过验证  
**基于**: Duan Lite v1.0 (Level 3)  
**目标**: 完整的面向对象支持，用于编写更强大的编译器

---

## 一、设计目标

Level 4 在 Level 3（Duan Lite）的基础上增加完整的面向对象支持，使得：
1. 可以定义包含属性和方法的类
2. 可以通过 `己` 访问对象属性和调用方法
3. 支持类继承和方法重写
4. 支持 `父` 关键字调用父类方法（映射为 `super()`）
5. 编译器可以自举（用 Level 4 编译自身，结果一致）

---

## 二、Level 3 → Level 4 变更清单

### 2.1 新增语法特性

| 特性 | 说明 |
|------|------|
| 类定义 | `类 类名：... 结束。` |
| 属性声明 | `属性 属性名。`（文档用途，运行时动态创建） |
| 属性访问表达式 | `己.x`、`obj.attr`、`a.b.c`（多层） |
| 属性赋值 | `设 己.x 为 值。`、`设 obj.attr 为 值。` |
| 方法定义 | `段落 方法名 接收 己, 参数...：` |
| 方法调用表达式 | `obj.method(args)`、链式调用 `a.b().c()` |
| 类继承 | `类 子类(父类)：` |
| `父` 关键字 | `父.method(args)` → `super().method(args)` |
| 下划线标识符 | 支持 `__init__`、`my_var` 等含下划线的标识符 |

### 2.2 新增关键字

在 Level 3 基础上新增 5 个关键字：

| 关键字 | 用途 |
|--------|------|
| `类` | 类定义 |
| `属性` | 属性声明（可选，文档用途） |
| `己` | 类方法中的 self 参数 |
| `父` | 调用父类方法（super()） |
| `继承` | 预留关键字（当前使用括号语法） |

**Level 4 关键字总数**: 34 个

### 2.3 词法分析变更

- **点号 `.`**：从"直接跳过"改为"识别为独立 token（DOT）"
  - 用于属性访问和方法调用
- **下划线 `_`**：新增为标识符合法字符
  - 支持 `__init__`、`my_var`、`_private` 等命名

### 2.4 语法解析变更

- 表达式层级增加：**成员访问表达式**（优先级最高，高于函数调用）
- `comp_set` 支持属性赋值：`设 己.x 为 值`、`设 obj.attr 为 值`
- `compile_class` 支持类定义、继承语法、方法编译
- `compile_block` / `compile_stmts` 支持成员调用语句：`obj.method()`
- 新增 `成员表达式` 解析函数

### 3.1 新增 Token 类型

| Token 类型 | 示例 | 说明 |
|-----------|------|------|
| `DOT` | `.` | 点号，用于属性访问 |

### 3.2 关键字（Level 4，共 32 个）

在 Level 3 的 29 个关键字基础上新增 3 个：

| 类别 | 关键字 |
|------|--------|
| 变量 | `设`、`为` |
| 函数 | `段`、`段落`、`接收`、`返回` |
| 控制流 | `如果`、`否则`、`当`、`结束` |
| 循环 | `遍历`、`在` |
| 算术 | `加`、`减`、`乘`、`除`、`取模` |
| 比较 | `等于`、`不等于`、`小于`、`大于`、`小于等于`、`大于等于` |

- 不再跳过点号

## 四、语法规范

### 4.1 类定义

**语法**：
```
类 类名：
  属性 属性名。
  ...
  段落 方法名 接收 己, 参数1, 参数2, ...：
    ...
  结束。
结束。
```

**带继承的语法**：
```
类 子类名(父类名)：
  ...
结束。
```

**示例**：
```
类 Point：
  属性 x。
  属性 y。

  段落 __init__ 接收 己, x, y：
    设 己.x 为 x。
    设 己.y 为 y。
  结束。

  段落 norm 接收 己：
    返回 己.x 乘 己.x 加 己.y 乘 己.y 的平方根。
  结束。
结束。
```

### 4.2 属性访问

**语法**：
```
表达式 . 标识符
```

**示例**：
```
设 p 为 Point(3, 4)。
设 a 为 p.x。      # 属性读取
设 己.x 为 10。   # 属性赋值（仅在方法内有效）
```

### 4.3 方法调用

**语法**：
```
表达式 . 标识符(参数列表)
```

**示例**：
```
设 p 为 Point(3, 4)。
设 d 为 p.norm()。     # 调用方法
p.move(1, 2)           # 调用有参方法
```

### 4.4 继承

**语法**：
```
类 子类名(父类名)：
  ...
结束。
```

**使用 `父` 调用父类方法**：
```
类 子类名(父类名)：
  段落 __init__ 接收 己, x, y, z：
    父.__init__(己, x, y)   # 调用父类构造
    设 己.z 为 z。
  结束。
结束。
```

## 五、表达式层级（Level 4）

在 Level 3 基础上，增加成员访问层级（优先级最高）：

| 优先级 | 层级 | 运算符 | 说明 |
|--------|------|--------|------|
| 1（最高） | 成员访问 | `.` | 属性访问、方法调用 |
| 2 | 一元 | `非`、函数调用 | 前缀运算符 |
| 3 | 乘除 | `乘`、`除`、`取模` | 乘除类 |
| 4 | 加减 | `加`、`减` | 加减类 |
| 5 | 比较 | `等于`、`不等于`、`小于`、`大于`、`小于等于`、`大于等于` | 比较运算 |
| 6 | 逻辑与 | `且` | 布尔与 |
| 7（最低） | 逻辑或 | `或` | 布尔或 |

### 5.1 成员访问表达式

**产生式**：
```
成员表达式 → 一元表达式 ( "." 标识符 )*
```

**注意**：
- 方法调用是成员访问 + 函数调用的组合：`obj.method(args)` = `(obj.method)(args)`
- 即先通过成员访问获取方法引用，再进行函数调用

## 六、代码生成映射

### 6.1 类定义

**Duan**：
```
类 类名：
  属性 a。
  属性 b。
  段落 __init__ 接收 己, a, b：
    设 己.a 为 a。
    设 己.b 为 b。
  结束。
结束。
```

**Python**：
```python
class 类名:
    def __init__(self, a, b):
        self.a = a
        self.b = b
```

### 6.2 继承

**Duan**：
```
类 子类(父类)：
  ...
结束。
```

**Python**：
```python
class 子类(父类):
    ...
```

### 6.3 属性访问

**Duan**：`己.x`  
**Python**：`self.x`

**Duan**：`obj.attr`  
**Python**：`obj.attr`

### 6.4 属性赋值

**Duan**：`设 己.x 为 值。`  
**Python**：`self.x = 值`

**Duan**：`设 obj.attr 为 值。`  
**Python**：`obj.attr = 值`

### 6.5 方法调用

**Duan**：`obj.method(a, b)`  
**Python**：`obj.method(a, b)`

### 6.6 `父` 关键字

**Duan**：`父.method(己, args)`  
**Python**：`super().method(args)`

## 九、设计决策

### 9.1 为什么用点号而不是中文关键字？

**选择**：使用 `.` 作为属性访问运算符

**理由**：
- 程序员普遍熟悉点号语法（Python、Java、C# 等）
- 中文关键字用于语句级结构，点号用于表达式级访问，层次清晰
- 减少关键字数量，保持语言简洁

### 9.2 为什么 `己` 是显式参数？

**选择**：方法的第一个参数必须是 `己`

**理由**：
- 与 Python 的 `self` 一致，实现简单
- 显式优于隐式，更易理解
- 编译器实现简单，不需要特殊处理

### 9.3 为什么支持单继承？

**选择**：只支持单继承，不支持多继承

**理由**：
- 简单性：单继承实现简单，语义清晰
- 足够性：对于编译器开发，单继承已经足够
- 可扩展性：未来 Level 5 可以考虑增加多继承或 Mixin

### 9.4 `父` vs `super()`

**选择**：提供 `父` 关键字，同时支持 `父.方法名(己, args)` 语法

**理由**：
- 中文命名更统一
- `父` 比 `super` 更直观
- 实现简单：直接映射到 `super()`

## 十、与 Level 3 的兼容性

- **完全向后兼容**：所有 Level 3 语法在 Level 4 中仍然有效
- **新增点号 token**：Level 3 中点号被跳过，Level 4 中被识别为 DOT
  - 这意味着 Level 3 中字符串外的点号会被忽略，而 Level 4 中会有意义
  - 但 Level 3 代码中不应该有点号（因为没有属性访问语法），所以不影响
- **新增关键字**：`继承`、`父`
  - Level 3 代码中如果使用这些词作为变量名，会在 Level 4 中变成关键字
  - 但 Level 3 是可信核心，代码中不会使用这些词作为标识符

## 十一、编译器优化

### 11.1 优化目标

在保持功能完整和自举能力的前提下：
- 减少编译器源码体积
- 减少生成代码的体积
- 提升编译速度（权衡取舍）

### 11.2 已实施优化

| 优化项 | 说明 | 源码减少 | 生成代码减少 |
|--------|------|----------|--------------|
| 关键字列表化 | 将 31 个链式 if 替换为列表+循环查找 | ~1.5KB | ~2KB |
| 成员链提取 | 提取 `处理成员链` 公共函数，消除 `己`/`父`/`成员表达式` 三处重复逻辑 | ~1KB | ~0.5KB |
| 代码压缩增强 | 行尾空白移除 + 连续空行限制为1个 | - | ~0.3KB |
| **总计** | | **~6.3%** | **~7.6%** |

### 11.3 性能数据

编译 `bootstrap_level4.duan`（28,528 字符）：

| 指标 | 优化前 | 优化后 | 变化 |
|------|--------|--------|------|
| 编译器源码 | 30,435 字符 | 28,528 字符 | -6.3% |
| 生成代码 | 34,034 字符 | 31,090 字符 | -7.6% |
| 编译时间 | ~0.086 秒 | ~0.107 秒 | +24% |
| 压缩比 | 1.12x | 1.09x | 更紧凑 |

**说明**：编译速度略有下降（约 24%），主要因为：
1. 关键字查找从平均 10 次比较变为平均 15 次列表遍历
2. 代码压缩增加了行尾空白修剪的开销

对于自举编译器而言，代码紧凑性和可维护性的提升优先于微小的速度损失。
