## 1. 概述

### 1.1 目标

创建一个交互式开发环境（REPL），支持：
- 逐行输入段言代码并立即执行
- 命令历史记录和多行代码输入
- 自动补全和语法高亮（可选增强）
- 调试支持（断点、单步执行）

### 1.2 目标用户

- **初学者**：简单易懂的界面，丰富的帮助提示
- **开发者**：高级功能（自动补全、语法高亮、调试）

### 1.3 设计原则

- **轻量核心**：无第三方依赖也可运行
- **可选增强**：安装 `prompt_toolkit` 后启用高级功能
- **混合执行**：简单表达式解释执行，复杂代码编译执行

## 3. 混合执行引擎

### 3.1 执行流程

```
用户输入 → 语法分析 → 判断复杂度 → 选择执行路径
                         ↓
            ┌────────────┴────────────┐
            ↓                         ↓
      简单表达式                  复杂代码块
      (单行、无定义)              (段落/类定义)
            ↓                         ↓
      解释执行                   编译执行
      (AST遍历)                  (生成Python)
            ↓                         ↓
      直接返回                   exec(py_code)
            ↓                         ↓
            └────────────┬────────────┘
                         ↓
                    输出结果
```

### 3.2 复杂度判断规则

| 类型 | 执行方式 | 示例 |
|------|----------|------|
| 变量声明 | 编译执行 | `设 甲 为 3。` |
| 表达式 | 解释执行 | `甲 加 5` |
| 段落定义 | 编译执行 | `段落 平方...` |
| 类定义 | 编译执行 | `类 狗...` |
| 条件语句 | 编译执行 | `如果...` |
| 函数调用 | 解释执行 | `打印(甲)` |

## 4. REPL命令

### 4.1 内置命令

| 命令 | 中文别名 | 功能 |
|------|----------|------|
| `help` | `帮助` | 显示帮助信息 |
| `exit` | `退出` | 退出REPL |
| `clear` | `清除` | 清屏 |
| `reset` | `重置` | 重置环境 |
| `vars` | `变量` | 显示所有变量 |
| `funcs` | `段落` | 显示所有段落 |
| `classes` | `类` | 显示所有类 |
| `history` | `历史` | 显示命令历史 |
| `load <file>` | `加载` | 加载并执行文件 |
| `save <file>` | `保存` | 保存当前会话 |
| `debug on/off` | `调试` | 开启/关闭调试模式 |
| `step` | `单步` | 单步执行（调试模式） |
| `break <line>` | `断点` | 设置断点 |

### 4.2 特殊语法

- 以 `:` 开头为命令（如 `:help`）
- 以 `#` 开头为注释
- 空行跳过
- 多行以 `结束。` 或空行结束

## 5. 增强功能

### 5.1 自动补全

补全来源：
- **关键字**：设、为、段落、类、如果、否则、返回...
- **动词白名单**：打印、长、首、末、排序、求和...
- **内置类型**：数、串、列、典、布尔...
- **当前环境变量**：用户定义的变量/段落/类名
- **模块符号**：已导入的符号

触发方式：
- `Tab` 键触发补全
- 连续 `Tab` 显示所有选项

### 5.2 语法高亮

颜色方案：
| 类型 | 颜色 | 示例 |
|------|------|------|
| 关键字 | 蓝色 | `段落`、`类`、`如果` |
| 动词 | 绿色 | `打印`、`排序` |
| 字符串 | 黄色 | `"你好"` |
| 数字 | 紫色 | `42`、`3.14` |
| 注释 | 灰色 | `# 注释` |
| 错误 | 红色 | 语法错误位置 |

## 6. 调试支持

### 6.1 调试模式

**启动调试**：
```
段言> :debug on
调试模式已开启

段言> 设 甲 为 3。
[调试] 执行: 变量声明 '甲' = 3
[调试] 环境: {'甲': 3}

段言> 打印(甲)。
[调试] 执行: 函数调用 '打印'
[调试] 参数: [3]
3
```

### 6.2 断点设置

```
段言> :load test.duan
加载文件: test.duan

段言> :break 5
断点设置: 行5

段言> :run
执行到断点 行5:
  设 总和 为 0。
[调试] 当前环境: {'列表': [1,2,3]}
暂停执行，输入 :step 继续
```

### 6.3 调试命令

| 命令 | 功能 |
|------|------|
| `:step` | 单步执行下一行 |
| `:continue` | 继续执行到下一个断点 |
| `:inspect <var>` | 查看变量详情 |
| `:backtrace` | 显示调用栈 |
| `:watch <var>` | 监视变量变化 |

## 7. 入口方式

### 7.1 命令行启动

```bash
# 独立REPL入口
python duan_repl.py

# 或通过CLI
duan repl
```

### 7.2 统一CLI整合

```bash
# 无参数时自动进入REPL
duan

# 其他命令
duan compile <file>    # 编译
duan run <file>        # 执行
duan parse <file>      # 解析
```

## 8. 依赖管理

### 8.1 核心依赖（必需）

- Python 3.8+
- antlr4-python3-runtime
- 项目内部模块（lexer、parser、code_generator）

### 8.2 增强依赖（可选）

- `prompt_toolkit` >= 3.0：自动补全、语法高亮
- `pygments`：高级语法高亮（可选）

安装方式：
```bash
pip install prompt_toolkit pygments
```
