# 段言（Duan）项目结构说明

## 架构总览

段言的编译器采用**流水线架构**，从源代码到执行/编译经过五个阶段：

```
源文件 (.duan) 
    │
    ▼
词法分析器 (Lexer) ──── 三层分词机制
    │
    ▼
语法解析器 (Parser) ──── 递归下降 + 元数驱动
    │
    ▼
语义分析器 (Analyzer) ── 类型检查 + 作用域分析
    │
    ▼
代码生成器 (Generator) ── 双后端：Python | LLVM
    │
    ▼
目标代码 / 执行
```

## 核心模块详解（src/）

### 1. 词法分析阶段

#### src/lexer.py — 词法分析器
- **职责**：将源代码字符串转换为 Token 流
- **核心机制**：`决策29` 三层分词
  1. **类型切换自动分词** — 不同字符类型（汉字、字母、数字）自动切分
  2. **双字关键词优先匹配** — 最长匹配原则，优先匹配双字关键字
  3. **预扫描用户定义** — 先扫描用户定义的变量/函数名，避免被错误拆分
- **支持**：无空格连续书写、中文数字（一～十）、各种注释（`#`、`//`、`注`）
- **依赖**：[tokens.py](file:///g:/dumategithub/duan/src/tokens.py)、[keywords.py](file:///g:/dumategithub/duan/src/keywords.py)

#### src/tokens.py — Token 类型定义
- **职责**：定义所有 Token 类型和数据结构
- **Token 类型**：NUMBER、STRING、IDENTIFIER、KEYWORD、各种符号等

#### src/keywords.py — 关键字和动词元数
- **职责**：定义所有关键字和动词的参数数量
- **核心设计**：
  - `决策27`：双字关键字（`定义`、`如果`、`段落`、`结束` 等）
  - `决策28`：元数驱动解析（`VERB_ARITY` 字典声明每个动词的参数数量）

### 2. 语法解析阶段

#### src/duan_parser_v3.py — 递归下降解析器
- **职责**：将 Token 流解析为 AST（抽象语法树）
- **核心设计**：
  - 递归下降解析算法
  - 元数驱动参数收集（动词声明参数数量，自动收集参数）
  - 支持完整 v1.6 语法
- **支持的语法结构**：
  - 变量声明：`定义 X 等于 Y。`、`设 X 为 Y。`
  - 条件语句：`如果...那么...否则...结束。`
  - 循环：`当...：...结束。`、`遍历 X 之 Y：...结束。`
  - 段落定义：`段落 名 接收 参数：...结束。` 或 `《名》段(参数)：`
  - 类定义：`类 名 [继承 父类]：...结束。`
  - 异常处理：`尝试：...捕获：...最终：...结束。`
  - 导入/导出：`导入 模块名。`、`从 模块 导入 符号。`、`导出 符号。`
  - 赋值：`标识符 等于 值。`
  - Self 赋值：`己属性 为 值。`
  - 管道操作：`数据 -> 处理 -> 输出`

#### src/class_parser.py — 类定义解析器扩展
- **职责**：类定义的补充解析逻辑
- **支持**：属性声明、构造方法、参数列表、类型注解

### 3. AST 节点定义

#### src/ast_nodes.py — AST 节点定义（手写解析器版）
- **职责**：定义手写解析器使用的 AST 节点类型
- **节点类型**：约 30+ 种，涵盖字面量、表达式、语句、定义等

#### src/ast_unified.py — AST 节点定义（统一版）
- **职责**：统一的 AST 节点定义，兼容手写解析器和 ANTLR 解析器
- **新增节点**：ListLiteral、DictLiteral、TryStatement、ThrowStatement、NewExpression 等
- **优势**：支持双后端（Python + LLVM）的统一 AST 格式

### 4. 语义分析

#### src/semantic_analyzer.py — 语义分析器
- **职责**：检查 AST 的语义正确性
- **功能**：
  - 符号表管理（作用域嵌套）
  - 变量重复定义检测
  - 段落/函数未定义检测
  - 类型一致性检查

#### src/type_inferencer.py — 类型推断器
- **职责**：基于 AST 进行类型推断
- **功能**：
  - 基本类型推断（数字、字符串、布尔、空值）
  - 变量类型追踪（跨作用域）
  - 二元运算类型推断（特别是 `+` 的字符串/数字歧义处理）
  - 函数调用返回类型推断

### 5. 代码生成

#### src/code_generator.py — Python 代码生成器
- **职责**：将 AST 转换为可执行的 Python 代码
- **功能**：
  - 变量声明 → `x = value`
  - 条件语句 → `if/else`
  - 循环 → `for/while`
  - 段落定义 → `def name(params):`
  - 类定义 → `class Name:`
  - 方法定义 → `def method(self, params):`
  - 运算符映射（中文 → Python 运算符）
  - 内置函数映射（`打印` → `print` 等）
  - 标准库自动导入

#### src/code_generator_unified.py — 统一版代码生成器
- **职责**：支持统一 AST 格式的代码生成
- **增强功能**：
  - 集成类型推断器，正确处理字符串连接和数字加法
  - 支持更多 AST 节点类型（ListLiteral、DictLiteral、NewExpression 等）
  - 支持 `elif` 链式条件

### 6. 模块系统

#### src/module_resolver.py — 模块解析器
- **职责**：模块查找、依赖分析、加载管理
- **功能**：
  - 模块搜索（多路径搜索：当前目录、配置路径、环境变量）
  - 依赖图构建（`DependencyGraph`）
  - 循环依赖检测（DFS 三色标记算法）
  - 拓扑排序（Kahn 算法，确定编译顺序）

### 7. 动词系统

#### src/verb_info.py — 动词信息模块
- **职责**：定义动词的元数和修改模式
- **核心设计**：
  - `决策34`：主谓/谓宾语义区分
    - `modify`：原地修改（`添加` → 列表.append()）
    - `functional`：返回新对象（`排序` → sorted(列表)）
    - `both`：两种模式都支持
  - Python 映射（`VERB_TO_PYTHON`）

### 8. REPL 交互环境

#### src/repl/ — REPL 子系统
- [core.py](file:///g:/dumategithub/duan/src/repl/core.py) — REPL 主循环：输入处理、多行支持、历史记录
- [executor.py](file:///g:/dumategithub/duan/src/repl/executor.py) — 代码执行器
- [commands.py](file:///g:/dumategithub/duan/src/repl/commands.py) — 命令处理器（`:help`、`:exit` 等）
- [completer.py](file:///g:/dumategithub/duan/src/repl/completer.py) — 自动补全（增强模式）
- [highlighter.py](file:///g:/dumategithub/duan/src/repl/highlighter.py) — 语法高亮（增强模式）

### 9. 核心基础设施

#### src/core/ — 核心基础设施
- [config.py](file:///g:/dumategithub/duan/src/core/config.py) — 编译器配置（输出格式、优化级别、调试选项等）
- [errors.py](file:///g:/dumategithub/duan/src/core/errors.py) — 错误类型层次（LexerError、ParserError、SemanticError、CodeGenError 等）
- [interfaces.py](file:///g:/dumategithub/duan/src/core/interfaces.py) — 抽象接口定义（ILexer、IParser、ISemanticAnalyzer、ICodeGenerator、ICompiler）

### 10. 标准库

#### src/stdlib/builtins.py — 内置函数
**文件 I/O**：读取文件、写入文件、追加文件、文件存在、目录存在等
**路径操作**：绝对路径、连接路径、目录名、文件名、扩展名
**系统函数**：环境变量、参数列表、退出程序、当前目录、执行命令
**字符串工具**：转整数、转浮点、转字符串、分割字符串、连接字符串、替换字符串
**列表工具**：列表追加、列表弹出、列表排序、列表反转、列表包含、列表创建
**字典工具**：字典创建、字典设置、字典删除、字典键列表、字典值列表
**类型检查**：是整数、是浮点、是字符串、是列表、是字典、是空

## ANTLR4 解析器（antlrparser/）

用于编译流程的 ANTLR4 实现，与手写解析器互补：

| 组件 | 职责 |
|------|------|
| DuanLang.g4 | ANTLR 语法定义文件（lexer + parser） |
| duan_parser/ | ANTLR 生成的 Python 解析器代码 |
| duan_ast.py | ANTLR 版本的 AST 节点定义 |
| duan_visitor.py | AST 访问器（ANTLR ParseTree → 自定义 AST） |
| duan_interpreter.py | 解释器（直接执行 AST） |
| duan_llvm.py | LLVM 代码生成器（编译为原生代码） |
| duan_tokenizer.py | 自定义分词器（ANTLR 兼容） |
| duan_compiler.py | 编译器主流程编排 |
| duan_cli.py | 命令行工具入口 |
| duan_repl.py | REPL 交互环境 |
