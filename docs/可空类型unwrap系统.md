# 段言可空类型 unwrap 系统

## 概述

段言实现了与 Swift/Kotlin/Rust 类似的可空类型安全机制。所有可能返回 `空` 的值必须被明确标记为可空类型，并在使用前显式解包（unwrap）。

**核心特性**：
- 🛡️ **编译期强制**：可空值参与运算/传递时必须先解包
- 🔓 **显式解包**：`值!` 或 `unwrap(值)` 强制断言非空
- 🧭 **类型推断**：解包后类型自动从 `数|空` 精化为 `数`
- ⚠️ **运行时断言**：解包失败时抛出带上下文的运行时错误
- 🔗 **HM 兼容**：与 HM 类型推断系统无缝协作

## 可空类型（Optional）

### 类型层次

```
数             ← 非空，可直接参与运算
数|空          ← 可空，必须先解包

串             ← 非空
串|空          ← 可空

ClassType('向量')       ← 非空对象
ClassType('向量')|空    ← 可空对象
```

### 表示方式（内部）

```python
# 类型系统内部表示
OptionalTypeWrapper(inner_type=NumberType())  # 数|空
OptionalTypeWrapper(inner_type=StringType())  # 串|空

# 类型判断方式
if value_type._type_id == TYPE_ID_OPTIONAL:
    # 这是一个可空类型
    inner = value_type.inner_type
```

### 可空类型的来源

1. **返回 `空` 的段落调用**：`获取用户(id)` 可能返回 `空`
2. **条件语句赋值**：`如果 条件 那么: x = 五 否则: x = 空。`
3. **显式类型声明**：`定义 值 为数|空。`
4. **从外部模块导入**：外部函数可能返回可空值

## 解包运算符

### 后置 `!` 运算符（推荐）

```段言
定义 值 等于 获取可能空的数据()。
打印 值!。        ← 断言非空后解包
```

**代码生成**：

```python
值 = 获取可能空的数据()
_duan_assert_not_none(值, "解包空值")   # 运行时断言
打印(值)
```

**类型推断**：

```
值: 数|空        ← 推断为可空
值!: 数          ← 解包后类型精化为数
```

### `unwrap(值)` 函数调用形式

```段言
打印 unwrap(值)。
```

功能与 `!` 等价，可读性更强，适合嵌套表达式。

### 安全访问（`?.` 链式）

```段言
定义 对象 等于 获取可能空的对象()。
如果 对象 != 空:
    打印 对象.属性。
结束。
```

## 强制检查机制

### 1. 运算时检查

所有需要非空值的运算都会在推断阶段检查：

```段言
定义 x 等于 获取值()。   ← 推断为 数|空
定义 和 等于 x 加 三。   ← 错误: 可空值不可直接参与运算
                            提示: 请先使用 x! 解包
```

**推断逻辑**：

```python
# 在 type_inferencer 的 BinaryOp 推断中
left_type = self._infer_expr(expr.left)  # 数|空
if left_type._type_id == TYPE_ID_OPTIONAL:
    self.errors.append(f"运算左侧为可空类型，请先解包: {left_type}")
```

### 2. 段落参数检查

调用段时，非可空形参不能接受可空实参：

```段言
段落 打印(消息):
    显示 消息。
结束。

定义 消息 等于 可能空()。
打印(消息)。     ← 错误: 形参是 串，实参是 串|空
打印(消息!)。    ← 正确: 先解包
```

### 3. 明确允许的可空操作

以下操作允许可空值：
- ✅ 与 `空` 比较：`如果 值 == 空: ...`
- ✅ 赋值给可空变量
- ✅ 作为可空参数传递给段
- ✅ 在 `如果` 语句的真值分支中自动解包（未来版本）

## UnwrapExpression AST 节点

### 节点定义（src/ast_nodes.py）

```python
@dataclass
class UnwrapExpression(ASTNode):
    """解包表达式：值! 或 unwrap(值)"""
    value: Any = None          # 被解包的表达式
```

### 解析过程（src/parser_expr.py）

1. 解析任意表达式 `expr`
2. 检查下一个 token 是否是 `!` 或 `！`
3. 若是，包装为 `UnwrapExpression(expr)`
4. 支持 `unwrap(expr)` 函数形式

### 代码生成（src/code_generator.py）

```python
def _gen_unwrap(self, expr):
    inner = self._gen_expr(expr.value)
    return f"_duan_unwrap({inner})"
```

生成的 Python 代码：

```python
def _duan_unwrap(_x):
    assert _x is not None, "尝试解包空值"
    return _x

# 使用
_duan_unwrap(值)
```

## 与 HM 推断系统的交互

### 示例 1：解包改变推断

```段言
段落 获取():
    返回 空。        ← 实际返回类型是 空
结束。

定义 x 等于 获取()。   ← x: 空|空 ≈ 数|空（取决于注释）
定义 y 等于 x!。       ← y: 数  ← 解包后恢复为非空
```

**推断流程**：

1. `获取()` 的返回类型推断为 `空`（或标注类型）
2. `x` 的类型是 `空`
3. `x!` 通过 `UnwrapExpression`，推断为 `空` 的非空版本（取决于类型推断上下文）
4. HM 合一后得到最终类型

### 示例 2：多态 + 可空

```段言
段落 或默认(值, 默认):
    如果 值 == 空:
        返回 默认。
    否则:
        返回 值!。
    结束。
结束。

# 调用
或默认(可能空(), 五)    ← 返回类型: 数
```

### 示例 3：与泛型交互

```段言
段落 安全获取(列表, 索引):
    如果 索引 大于 列表长度:
        返回 空。
    否则:
        返回 列表[索引]。
    结束。
结束。

# 推断结果
安全获取: (列表[T], 数) -> T|空
```

## 运行时行为

### 解包成功（非空值）

```段言
定义 值 等于 五。
显示 值!。  ← 正常输出 "5"
```

```python
# 生成代码
值 = 5
assert 值 is not None, "尝试解包空值"
显示(值)
```

### 解包失败（空值）

```段言
定义 值 等于 空。
显示 值!。  ← 抛出 AssertionError: 尝试解包空值
```

```python
# 生成代码
值 = None
assert 值 is not None, "尝试解包空值"  # 在这里失败
显示(值)
```

## 推荐编程实践

### 1. 尽量保持非空

段言默认非空，可空是显式声明的。优先设计函数返回非空值。

```段言
段落 获取年龄():
    返回 二十五。    ← 返回 数
结束。
```

### 2. 提前解包，避免在表达式中反复解包

```段言
# 好
定义 年龄 等于 获取可能空的年龄()。
如果 年龄 == 空:
    返回 零。
结束。
# 之后可以安全使用 年龄!

# 不好（但允许）
计算(获取可能空的年龄(), 其他值!)
```

### 3. 为可空值提供默认值

```段言
段落 或默认(值, 默认):
    如果 值 == 空:
        返回 默认。
    否则:
        返回 值!。
    结束。
结束。
```

## 未来路线图

1. **`?.` 安全调用**：`对象?.属性` 在对象为空时返回空
2. **`??` 默认值运算符**：`值 ?? 默认值` 提供解包后的默认值
3. **`如果 让` 绑定**：在条件分支中自动将变量视为非空
4. **`匹配空` 模式**：`匹配 值: 情况 空: ... 情况 x: ...` 完整模式匹配
5. **`可空链传播`**：整个链式调用中只要有一个为空，结果为空
