## 一、自举定义

### 1.1 什么是真正的自举？

**自举（Bootstrap）**：用语言自身编写编译器，能够编译自己。

**三个层次**：
1. **伪自举** - 代码中大量使用Python代码块（如言语言当前状态）
2. **部分自举** - 核心逻辑用段言实现，少量依赖Python内置函数
3. **完全自举** - 编译器完全用段言实现，能编译自己

### 1.2 我们的目标

**目标**：实现**部分自举**
- 编译器核心逻辑用段言实现
- 依赖标准库提供的文件I/O等基础函数
- 能够编译一个简化版的段言程序
- 能够编译自己（完整版）

## 三、技术挑战和解决方案

### 3.1 当前限制

| 限制 | 影响 | 解决方案 |
|------|------|---------|
| 无完善的字符串处理 | 难以实现字符串操作 | 扩展标准库，添加字符串函数 |
| 无列表推导式 | 难以创建复杂列表 | 使用循环+追加 |
| 无类系统 | 难以组织复杂逻辑 | 使用字典+函数式风格 |
| 无异常处理 | 难以优雅处理错误 | 使用返回值+检查 |

### 3.2 数据结构设计

**Token表示**（使用字典）：
```段言
定义令牌等于字典创建。
字典设置参数令牌，"类型"，"标识符"。
字典设置参数令牌，"值"，"变量名"。
字典设置参数令牌，"行"，1。
字典设置参数令牌，"列"，5。
```

**AST节点表示**（使用字典）：
```段言
定义节点等于字典创建。
字典设置参数节点，"类型"，"变量声明"。
字典设置参数节点，"名称"，"甲"。
字典设置参数节点，"值"，值节点。
```

### 3.3 函数式风格

由于没有类，使用函数式风格：

```段言
# 创建Token
《创建令牌》段(类型, 值, 行, 列)：
  定义令牌等于字典创建。
  字典设置参数令牌，"类型"，类型。
  字典设置参数令牌，"值"，值。
  字典设置参数令牌，"行"，行。
  字典设置参数令牌，"列"，列。
  返回令牌。

# 使用Token
定义tok等于《创建令牌》参数"标识符"，"甲"，1，5。
定义类型等于字典获取参数tok，"类型"。
```

## 四、模块设计

### 4.1 词法分析器（lexer.duan）

**接口**：
```段言
导出《词法分析》，《创建令牌》。

《词法分析》段(源代码)：
  # 返回令牌列表
  定义令牌列表等于列表创建。
  # ... 分析逻辑
  返回令牌列表。
```

**支持的Token类型**：
- 关键字：定义、等于、如果、那么、否则、返回、段
- 标识符：中文变量名、英文变量名
- 数字：整数、浮点数、中文数字
- 字符串：双引号字符串
- 符号：。，（）：【】

### 4.2 语法分析器（parser.duan）

**接口**：
```段言
导出《语法分析》，《解析变量声明》，《解析函数定义》。

《语法分析》段(令牌列表)：
  # 返回AST
  定义ast等于字典创建。
  # ... 解析逻辑
  返回ast。
```

**支持的AST节点**：
- 模块（Module）：语句列表
- 变量声明（VarDecl）：名称、值
- 函数定义（FuncDef）：名称、参数、体
- 条件语句（IfStmt）：条件、真分支、假分支
- 表达式（Expr）：类型、值、操作符

### 4.3 代码生成器（codegen.duan）

**接口**：
```段言
导出《生成代码》，《生成变量声明》，《生成函数定义》。

《生成代码》段(ast)：
  # 返回Python代码字符串
  定义代码等于空字符串。
  # ... 生成逻辑
  返回代码。
```

### 4.4 编译器（compiler.duan）

**接口**：
```段言
导出《编译》，《编译文件》。

《编译》段(源代码)：
  定义令牌列表等于《词法分析》参数源代码。
  定义ast等于《语法分析》参数令牌列表。
  定义代码等于《生成代码》参数ast。
  返回代码。

《编译文件》段(输入路径, 输出路径)：
  定义源代码等于读取文件参数输入路径。
  定义代码等于《编译》参数源代码。
  写入文件参数输出路径，代码。
  打印"编译完成"。
```

## 五、实现计划

### 5.1 第一阶段：简化版词法分析器（1-2天）

**目标**：实现 `lexer_mini.duan`

**功能**：
- 识别标识符（中文/英文）
- 识别关键字（定义、等于、如果、返回等）
- 识别数字（整数）
- 识别符号（句号、逗号、括号）

**测试**：
```段言
定义源代码等于"定义甲等于123。"。
定义令牌列表等于《词法分析》参数源代码。
打印令牌列表。
```

### 5.2 第二阶段：简化版语法分析器（1-2天）

**目标**：实现 `parser_mini.duan`

**功能**：
- 解析变量声明
- 解析简单的表达式
- 构建AST

**测试**：
```段言
定义源代码等于"定义甲等于123。"。
定义令牌列表等于《词法分析》参数源代码。
定义ast等于《语法分析》参数令牌列表。
打印ast。
```

### 5.3 第三阶段：简化版代码生成器（1天）

**目标**：实现 `codegen_mini.duan`

**功能**：
- 生成变量声明代码
- 生成简单的Python代码

**测试**：
```段言
定义源代码等于"定义甲等于123。"。
定义代码等于《编译》参数源代码。
打印代码。
# 输出: 甲 = 123
```

### 5.4 第四阶段：整合测试（1天）

**目标**：验证 `compiler_mini.duan`

**测试用例**：
1. 简单变量声明
2. 简单函数定义
3. 简单条件语句

### 5.5 第五阶段：完整版实现（3-5天）

**目标**：实现完整版编译器

**扩展功能**：
- 完整的词法分析（字符串、中文数字等）
- 完整的语法分析（循环、模块、类等）
- 完整的代码生成（所有语法）

### 5.6 第六阶段：自举验证（1天）

**目标**：验证自举成功

**验证步骤**：
```bash
# 1. 用Python版编译器编译段言编译器
python cli/duanc.py bootstrap/compiler.duan -o bootstrap/compiler_gen.py

# 2. 用生成的Python代码再次编译
python bootstrap/compiler_gen.py bootstrap/compiler.duan -o bootstrap/compiler_gen2.py

# 3. 比较两次输出
diff bootstrap/compiler_gen.py bootstrap/compiler_gen2.py
# 应该相同或等价
```
