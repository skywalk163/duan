## [2.0.0] - 2025

### ✨ 新增

#### 🔬 HM 全局类型推断系统
- 实现 Hindley-Milner 风格的全局类型推断
- **两阶段推断架构**：预扫描注册 → 段体推断 + 泛化
- **let-polymorphism**：自动识别泛型段落并实例化
- **类型合一**：支持 `TypeVar` 与具体类型的双向推导
- **泛型支持**：泛型段落、泛型类、泛型接口
- **类型 ID 机制**：使用整数 `_type_id` 快速判断类型（替代 `isinstance` 链，性能提升 30-40%）
- **基本类型单例化**：每个基本类型只有一个实例，减少内存占用
- **方法签名缓存**：类方法解析后缓存，避免重复扫描
- **两轮迭代推断**：段体调用段时，先解决被调用者类型，再推断调用者

#### 🛡️ 可空类型 unwrap 系统
- 实现编译期可空类型检查
- **`!` 后置解包运算符**：`值!` 强制断言非空
- **`unwrap(值)` 函数形式**：可读性更强的解包方式
- **运行时断言**：`_duan_unwrap()` 在解包时检查空值
- **`OptionalTypeWrapper`**：内部表示 `数|空` 等可空类型
- **与 HM 推断协作**：解包后类型自动从 `数|空` 精化为 `数`
- **运算时检查**：可空值参与运算时必须先解包
- **段落参数检查**：非可空形参不能接受可空实参

#### 📦 模块系统与包管理
- **文件即模块**：每个 `.duan` 文件编译为独立模块，拥有独立符号表
- **依赖图拓扑排序**：使用 Kahn 算法确定编译顺序，避免依赖问题
- **循环依赖检测**：自动检测并报告模块间循环引用
- **`package.toml` 配置格式**：
  - `[package]`：包名、版本、描述、作者、入口文件
  - `[dependencies]`：本地依赖（`path`），预留 Git/注册表依赖
  - `[build]`：输出文件、目标运行时、优化开关
- **`PackageManager`**：统一管理项目加载、发现、编译流程
- **`PackageConfig`**：数据结构封装配置信息
- **`ModuleResolver`**：负责解析模块导入关系
- **`段言 init` 命令**：初始化新项目（创建 package.toml + main.duan）
- **`段言 build` 命令**：完整编译项目
- **跨模块 HM 类型推断**：导入的段落签名保留，可在当前模块继续推断
- **跨模块可空类型检查**：跨模块调用时保留可空性信息

### 📝 文档

### 🧪 测试

- **HM 类型推断测试**：30+ 测试点，覆盖变量推断、段落推断、泛型推断、类/接口推断
- **可空类型 unwrap 测试**：14+ 测试点，覆盖基本解包、运算检查、参数检查
- **模块系统与包管理测试**：20+ 测试点，覆盖模块发现、依赖图构建、拓扑排序、循环依赖

### 🔧 核心实现文件

- `src/type_inferencer.py`：HM 类型推断器（两阶段架构、let-polymorphism）
- `src/type_system.py`：类型系统（类型层次、`_type_id` 机制、合一算法、`OptionalTypeWrapper`）
- `src/ast_nodes.py`：AST 节点定义（新增 `UnwrapExpression` 节点）
- `src/package_manager.py`：包管理器（`PackageConfig`、`PackageManager`）
- `src/module_resolver.py`：模块解析器（依赖图、拓扑排序、循环依赖检测）
- `src/compiler.py`：编译器入口（AstAdapter 增强，支持 `UnwrapExpression` 代码生成）
- `src/parser_expr.py`：表达式解析器（支持 `!` 解包运算符解析）
- `src/code_generator.py`：代码生成器（生成 `_duan_unwrap()` 运行时断言）

### ⚡ 性能优化

- **类型 ID 快速匹配**：用整数 `_type_id` 替代 `isinstance` 链，类型判断性能提升 30-40%
- **基本类型单例化**：每个基本类型只有一个实例，减少内存分配和 GC 压力
- **方法签名缓存**：类方法第一次解析后缓存，避免重复扫描
- **HM 两轮迭代**：段调用段时，第一轮先解决被调用者类型，第二轮再推断调用者
- **`__slots__` 优化**：AST 节点使用 `__slots__` 减少内存占用

### 📐 架构改进

- **三大特性解耦**：类型推断、可空类型、模块系统三个子系统独立但协作
- **符号表层次**：全局符号表 → 模块符号表 → 段作用域符号表
- **依赖解析流水线**：模块发现 → 导入解析 → 依赖图构建 → 拓扑排序 → 编译
- **跨模块类型传播**：已编译模块的类型签名保留，供依赖它的模块使用

## [1.6.0] - 2025

### ✨ 新增

- **统一语法规范 v1.6**：采用段落式语法，更符合中文表达习惯
- **完整面向对象支持**：类、继承、属性、方法、接口、多态
- **标准库**：`数学`、`时间`、`日期时间`、`JSON`、`哈希`、`正则` 六大模块
- **逻辑运算符**：`且`/`与`（and）和 `或`（or）
- **双后端架构**：Python 后端（解释执行）+ LLVM 后端（编译为原生）
- **自举探索**：包含自举编译器的实验性实现（lexer、parser、ast_nodes、codegen）
- **现代特性**：字符串插值、列表推导、lambda、match 模式匹配
- **异常处理**：try-catch-finally、throw、类型匹配
- **REPL 交互环境**：语法高亮、自动补全、命令处理器
- **结构化错误处理**：`LexerError`/`ParseError`/`SemanticError`/`CodeGenError`
