## 技术决策

### 为什么选择自定义词法分析器 + 递归下降解析器？

| 方案 | 优点 | 缺点 |
|------|------|------|
| **自定义词法分析器 + 递归下降** | 完全控制解析逻辑<br>无空格分词<br>易于调试和扩展 | 需要自己实现<br>代码量较大 |
| Lark 集成 external_lexer | 利用 Lark 的 LALR 优化 | 集成复杂<br>Transformer 问题 |
| 纯 Lark regex lexer | 快速实现 | 无法处理无空格分词 |

**最终决策**: 自定义词法分析器 + 递归下降解析器（简单、直接、可控）

### 关键技术点

1. **三层分词机制**（决策29）
   - 类型切换自动分词：汉字/数字/符号边界
   - 双字关键词优先匹配：最长匹配算法
   - 中文数字识别：一～十自动转换

2. **性能优化**
   - 关键字按长度分组：O(k) 匹配，k 为最大关键字长度
   - 预编译字符类型判断：避免重复计算
   - 线性扫描：O(n) 时间复杂度

3. **递归下降解析**
   - 直观的语法规则映射
   - 易于扩展新语法结构
   - 清晰的错误定位

def parse_verb_call(self, verb: str) -> Tuple[ASTNode, int]:
    arity = get_arity(verb)
    
    if arity == -1:
        # 可变参数：收集到阻断符为止
        return self._collect_variable_args(verb)
    else:
        # 固定参数：收集指定数量
        return self._collect_fixed_args(verb, arity)

### semantic_identifier.py - 语义识别器（180行）

**功能**: 实现决策34，主谓/谓宾语义识别

**语义类型**:
```python
class SemanticType:
    SUBJECT_VERB = 'subject_verb'      # 主谓结构（原地修改）
    VERB_OBJECT = 'verb_object'        # 谓宾结构（返回新对象）
    ATTRIBUTE = 'attribute'            # 定语结构（临时生成）
    FUNCTIONAL = 'functional'          # 函数式调用
```

**识别规则**:
```python
# 规则1：主谓结构 → 原地修改
列表排序。 → 列表.sort()

# 规则2：谓宾结构 → 返回新对象
排序列表。 → sorted(列表)

# 规则3：定语结构 → 临时生成
排序后的列表。 → sorted(列表)
```

**Python代码生成**:
```python
def generate_python_code(semantic_type, verb, args, symbol_table):
    if semantic_type == SUBJECT_VERB:
        # 主谓结构：原地修改
        return f"{obj}.{method}({args})"
    elif semantic_type == VERB_OBJECT:
        # 谓宾结构：返回新对象
        return f"{function}({args})"
```
